Resource Hub Integration Guide

The Resource Hub serves as a central self-hosted Git platform for your files, notebooks, and models. You can synchronize resources natively using your command-line Git client or manage everything through the dynamic web interface.

Option A: Using the Web Upload Dashboard

If you do not want to use command-line commands, you can manage files directly from your browser.

1
Navigate to the Upload Tab Click the Upload Resource tab in the navigation bar above.
2
Select Files, Folders, or Drag-and-Drop Drag one or multiple files/folders into the drag area, or use the Choose Files / Choose Folder buttons. The web page preserves full nested folder path structures automatically.
3
Submit Commit & Push Provide a descriptive commit message, select your target branch, and click Commit & Push Web Upload. The server processes and pushes changes directly to the repository repository.
Pro-Tip: The drag-and-drop client automatically reads files recursively! You can drag an entire project folder, and it will keep the exact relative directory layout.

Option B: Using Native Command Line Git

You can interact with this repository using standard Git commands. All actions authenticate against your main platform login credentials.

1. Clone the repository

git clone {{ clone_url }}

2. Set up an existing local directory

git init
git remote add origin {{ clone_url }}
git add .
git commit -m "Initial commit"
git branch -M master
git push -u origin master

3. Push updates

git add .
git commit -m "Update resources"
git push origin master
Authentication Credentials: When prompted for a username and password by your local Git command line client, enter your own system platform username and password to authenticate.
